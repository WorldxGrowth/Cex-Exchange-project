## TECH STACK - WHY & WHAT

```
BACKEND:
├── Node.js + Express     → Fast, async, huge ecosystem
├── PostgreSQL            → Main database (reliable, ACID)
├── Redis                 → Order book cache, sessions, real-time
├── Socket.io             → WebSocket (live prices, orders)
├── Bull Queue            → Background jobs (deposit detect, bot orders)
├── JWT                   → Authentication tokens
└── bcrypt                → Password hashing

ADMIN PANEL:
└── React + Ant Design    → Professional admin UI (fast to build)

WEB FRONTEND:
├── React + TypeScript    → (tumhe aata hai VDSwap se)
├── TradingView Charts    → (tumhe aata hai)
└── Tailwind CSS          → Styling

ANDROID:
└── Flutter               → (tumhare paas DexPrime experience)

INFRASTRUCTURE:
├── Nginx                 → Reverse proxy
├── PM2                   → Node.js process manager
└── Let's Encrypt         → SSL
```

## TOKEN LISTING FLOW - Tera Sawal ✅

```
Token Owner aata hai:
        ↓
1. Exchange pe account create karta hai (normal user)
        ↓
2. "Token Listing" apply karta hai
   → Token details submit karta hai
   → Listing fee pay karta hai (USDT)
        ↓
3. Apne token contract address deta hai
   (jo usne BSC/ETH/VDChain pe deploy kiya)
        ↓
4. Liquidity deta hai:
   → Usee apne exchange account mein
     TOKEN deposit karta hai
   → USDT bhi deposit karta hai
        ↓
5. Admin approve karta hai
   → Trading pair create hota hai
   → Market making bot start hota hai
   → Scheduled date pe live ho jaata hai
        ↓
6. Sab kuch database mein store ✅
   token_listings table → application
   user_wallets table   → uska deposit
   trading_pairs table  → pair
   market_making_bots   → bot config
```
